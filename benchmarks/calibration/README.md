# Topos Calibration Evaluations

This directory contains the infrastructure and data for calibrating Topos metric thresholds against real-world, open-source codebases across multiple ecosystems.

## Overview

The calibration suite validates two core properties of the Topos evaluation model:
1. **Distribution Fit:** Do the thresholds defined in the policy translators (Φᵢ) align with the natural distribution of metrics in high-quality, real-world code?
2. **Usage Alignment:** Do the metrics and their weighting under different UserPreferences profiles (simple, composable, secure) accurately capture the intended architectural usage of a package (e.g., as a reusable API library vs. a self-contained application)?

The canonical documentation for the experimental design, hypotheses, and detailed procedures is located in docs/calibration.md.

## Directory Structure

*   `scripts/`: Contains the executable logic for gathering evidence, running baseline evaluations, and analyzing scores.
    *   `run_structural_baseline*.py`: Runners for executing `topos evaluate` across the cohorts.
    *   `analyze_scores.py`: Aggregates the `structural_scores.jsonl` output, computes statistics, performs threshold sweeps, and cross-references data with usage labels and registry evidence.
    *   `collect_*_evidence.py`: Scripts to pull metadata signals from package registries (PyPI, crates.io, npm, vcpkg) to validate manual usage classifications.
    *   `refresh_top100_*.py`: Scripts to fetch the latest top-100 packages for each ecosystem.
*   `results/`: (Ignored by git) The output directory for the raw JSONL score dumps and the summarized `score_analysis.json` generated by the scripts.
*   `evidence/`: (Ignored by git) The output directory for the registry metadata collected by the `collect_*_evidence.py` scripts.
*   `top100_*.txt`: The static cohort lists of the top 100 packages for Python (PyPI), Rust (crates.io), JavaScript/TypeScript (npm), and C++ (vcpkg).
*   `usage_profiles.csv`: A deterministic, manual labeling of the Python cohort into `composable` (API-first) and `self_contained` (app-first) categories, used to test hypothesis alignment.

## Running the Calibrations

To run the full suite for a given ecosystem (e.g., Python):

1.  **Gather Evidence (Optional):** Cross-validate usage labels against registry metadata.
    ```bash
    python benchmarks/calibration/scripts/collect_pypi_evidence.py
    ```
2.  **Run Structural Baseline:** Evaluate the corpus.
    ```bash
    python benchmarks/calibration/scripts/run_structural_baseline.py
    ```
    This generates `benchmarks/calibration/results/structural_scores.jsonl`.
3.  **Analyze Scores:** Compute threshold statistics and pass rates.
    ```bash
    python benchmarks/calibration/scripts/analyze_scores.py --scores benchmarks/calibration/results/structural_scores.jsonl
    ```
    This prints a summary and generates `benchmarks/calibration/results/score_analysis.json`.

For detailed instructions on running multi-language calibrations (Rust, npm, vcpkg), refer to `docs/calibration.md`.

## v1.0.0 Threshold Findings

The initial Python calibration run revealed that the raw metric thresholds (`cyclomatic <= 15`, `max_func <= 10`, `entropy in [0.2, 0.8]`, `dangerous_calls == 0`, `taint_flows == 0`) are well-calibrated to the empirical distribution of the corpus. 

However, the fallback normalized score floors in `topos/evaluation/policies/base.py` required adjustment to mathematically align with the raw policy bounds:
*   **SIMPLE:** Adjusted to `0.40` (the exact score yielded by a file sitting on the raw `cyclomatic=15`, `max_func=10`, `entropy=0.2` boundary).
*   **SECURE:** Adjusted to `1.00` (to enforce the strict `count == 0` policy, as a single finding yields ~0.71 due to exponential decay).
*   **COMPOSABLE:** Remains `0.60` (aligns with the `fan_in / fan_out` cap of 15/40, which yields `0.625`).

See `docs/calibration.md` for the full write-up of these findings.
